# Plans

This directory holds the detailed implementation plans for items in
[`../QUEUE.md`](../QUEUE.md). Each plan is created with `/index` and corresponds
to exactly one queue item.

## Conventions

- **One file per queue item**, named `Q-NNN-<slug>.md` (matching the queue id).
- A plan is the *how*: goal, approach, files to touch, steps, risks, and how it
  will be verified. The queue is the *what* and *when*.
- When a plan is written, set its queue item to `📝 PLANNED` and link the file
  from the queue's `Plan` column.
- Plans are living docs — update them as the approach changes during work.
